# 對話服務與模型回退

> 文件使用原則：
> - 本文假設你正在使用 standalone `3klife-npc-brain` repo。
> - `<repo-root>` 代表你自己 clone 下來的專案根目錄。
> - 若本文包含啟動或驗證指令，預設以 Docker 為正式開發環境來源；本機 Python / venv 僅用於 IDE debug、LangGraph dev 或臨時工具。

> 說明 npc-brain runtime 對話 API、關鍵輸入維度、LLM provider fallback chain，以及 history cache 的用途。

## 這份文件適合什麼時候看

- dialogue 出現 fallback，但你不確定是在哪一層失敗
- 想理解 `providerTrace`、`qualityWarnings`、`repairUsed` 的含義
- 想知道 Cocos dev toolbar 的 model preset 到底在測什麼

本文件不展開：

- LangGraph Studio / deployment 操作
- Pinecone / Qdrant upsert 細節
- 人物資料推進流程本體

## API 入口

- `GET /healthz`
- `GET /v1/npc/context-options?generalId=<id>`
- `GET /v1/npc/keyword-options?generalId=<id>`
- `POST /v1/npc/dialogue`

`POST /v1/npc/dialogue` 會回傳：

- `text`
- `locale`
- `speechContextMode`
- `evidenceRefs`
- `usedKeywords`
- `rejectedKeywordKeys`
- `provider`
- `model`
- `providerTrace`
- `qualityWarnings`
- `repairUsed`

## 對話三個核心維度

### `locale`

預設 `zh-TW`，目前支援：

- `zh-TW`
- `en`
- `ja`

### `speechContextMode`

預設 `life_chat`，目前支援：

- `life_chat`
- `encounter_speech`
- `inner_monologue`
- `meeting_statement`

這個欄位是發話角度，不取代事件 `contextKey`。

### `llmModelPreset`

預設 `fallback_chain`，也可在開發時強制指定單一模型，例如：

- `gemini_pro`
- `gemini_flash`
- `gemini_flash_lite`
- `qwen2_5_7b`
- `qwen2_5_3b`
- `deepseek_r1_7b`
- `local_llama_env`

指定單一 preset 時，失敗就直接回錯，不會偷偷落回 deterministic template。

## 正式 fallback chain

建議順序：

1. `gemini`
2. `gemini_flash`
3. `gemini_flash_lite`
4. `local_llama`
5. `history_cache`
6. `deterministic`

### 各層用途

- `gemini`：主要高品質生成
- `gemini_flash`：處理高負載 / timeout / 配額壓力
- `gemini_flash_lite`：更輕量的備援
- `local_llama`：雲端不可用時的本地生成
- `history_cache`：回收既有 LLM 成功輸出
- `deterministic`：最後安全線

## 本地模型與 repair

本地 `local_llama` 預設對接 Ollama 相容 chat API，常用模型：

- `qwen2.5:7b`
- `qwen2.5:3b`

server 會加上：

- system + user messages
- JSON contract
- speaker / language / gibberish validator
- 一次 repair retry

因此回應中可能看到：

- `repairUsed = true`
- `qualityWarnings = ["repaired:..."]`

這代表第一輪品質不夠穩，但第二輪修正後通過。

## History Cache 是什麼？

每次 LLM 成功產出台詞後，service 會把摘要 append 到 `NPC_LLM_HISTORY_CACHE_PATH`，內容包含：

- `generalId`
- `contextKey`
- `locale`
- `speechContextMode`
- `keywordKeys`
- `evidenceRefs`
- `provider` / `model`
- `qualityWarnings`
- `repairUsed`
- `text`

它是開發期與 fallback 用的「既有成功台詞 queue」，不是手寫固定台詞表。

## 最小請求範例

```json
{
  "generalId": "zhang-fei",
  "contextKey": "changban-bridge",
  "selectedKeywordKeys": ["cao-cao", "serpent-spear"],
  "toneMode": "in-character",
  "locale": "zh-TW",
  "speechContextMode": "life_chat",
  "llmModelPreset": "fallback_chain",
  "maxChars": 90
}
```

## 健康檢查與 debug 建議

先看：

```bash
curl http://127.0.0.1:8765/healthz
```

若設定了 `NPC_LLM_DEBUG=1`，terminal 會額外輸出：

- `dialogue.build.start`
- `dialogue.build.resolved`
- `provider.request`
- `provider.response.raw`
- `provider.response.parsed`

最值得優先看的欄位：

- `providerTrace`
- `fallbackUsed`
- `generationMode`
- `repairUsed`
- `qualityWarnings`
