# 開發啟動與煙霧測試

> 文件使用原則：
> - 本文假設你正在使用 standalone `3klife-npc-brain` repo。
> - `<repo-root>` 代表你自己 clone 下來的專案根目錄。
> - 若本文包含啟動或驗證指令，預設以 Docker 為正式開發環境來源；本機 Python / venv 僅用於 IDE debug、LangGraph dev 或臨時工具。

> 本文中的 `<repo-root>` 代表你自己 clone 下來的專案根目錄。

這份文件描述 standalone `3klife-npc-brain` 的啟動與 smoke test。拆分後不要再從 `舊 monorepo 內嵌路徑` 啟動。

如果你只是要最快把服務跑起來，先看 [最短啟動路徑](./最短啟動路徑.md)。

## 首選：Docker Compose dev

```powershell
cd <repo-root>
docker compose -f docker-compose.dev.yml up -d --build
```

檢查 container：

```powershell
docker compose -f docker-compose.dev.yml ps
curl http://127.0.0.1:8765/healthz
```

停止：

```powershell
docker compose -f docker-compose.dev.yml down
```

## `.env` 最小集合

把 `.env.example` 複製為 `.env`，真實金鑰只放本機，不提交。

```powershell
cd <repo-root>
Copy-Item .env.example .env
```

常用欄位：

```text
NPC_BRAIN_DEPLOY_API_KEY=<shared-key>
GOOGLE_API_KEY=<key>
NPC_LLM_PROVIDER_ORDER=deterministic
PINECONE_API_KEY=<key>
NPC_PINECONE_INDEX=sanguo-npc-brain-dev
NPC_QDRANT_URL=http://host.docker.internal:6333
```

## Docker 內 smoke tests

```powershell
docker exec 3klife-npc-brain-dev python -B -m app.http_smoke_test
docker exec 3klife-npc-brain-dev python -B -m app.cocos_flow_smoke_test
docker exec 3klife-npc-brain-dev python -B -m app.llm_provider_smoke_test
docker exec 3klife-npc-brain-dev python -B -m app.vector_env_smoke_test
```

Sanguo governance：

```powershell
docker exec 3klife-npc-brain-dev python -B pipelines/sanguo-rag/validate_sanguo_governance.py --dry-run-report
docker exec 3klife-npc-brain-dev python -B pipelines/sanguo-rag/run_sanguo_governance_regression_harness.py --run-profile strict-local --no-write
```

## 可選：本機 Python 3.11

Docker 是正式開發來源；只有需要 IDE debug 時才建本機 venv。

```powershell
cd <repo-root>
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python -m uvicorn app.main:app --host 127.0.0.1 --port 8765 --reload
```

WSL2：

```bash
cd <repo-root>
python3.11 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
python -m uvicorn app.main:app --host 127.0.0.1 --port 8765 --reload
```

## LangGraph dev server

```powershell
cd <repo-root>
python -m pip install -U "langgraph-cli[inmem]"
langgraph dev --no-browser
```

## 常見問題

- `qdrantPing.reachable = false`：確認 Docker Desktop 與 Qdrant container 是否啟動。
- `pinecone.configured = false`：確認 `.env` 是否有 `PINECONE_API_KEY`。
- HTTP smoke 找不到 fixture：確認 `artifacts/data-pipeline/sanguo-rag` 已存在，或先重跑 pipeline 產生 artifacts。
