# 開發啟動與煙霧測試

> 給開發者的最短路徑：如何安裝、啟動 FastAPI / LangGraph、驗證 healthz、跑 smoke tests。

## 最短成功條件

如果你今天只是想知道「這台 npc-brain 有沒有活著」，先看這四件事：

1. `GET /healthz` 回 `200`
2. `app.smoke_test` 可以跑通
3. 至少有一條 provider 路徑可解釋（成功或合理 fallback）
4. `app.vector_env_smoke_test` 的輸出和你預期一致

這份文件的目標不是教部署，而是幫你快速判斷：**現在壞的是環境、provider、還是 artifact。**

## 一次性安裝

以下範例假設已在目標 venv 內執行；若還沒，先 `source` 對應的 `bin/activate`，或設定 `PYTHON_BIN` 指向該環境。若你偏好統一入口，可在 shell 先定義：

```bash
PYTHON_BIN="${PYTHON_BIN:-python}"
```

```bash
cd server/npc-brain
"$PYTHON_BIN" -m pip install -r requirements.txt
```

若要跑 LangGraph：

```bash
"$PYTHON_BIN" -m pip install -U "langgraph-cli[inmem]"
```

## `.env` 最小集合

依你要測的東西不同，至少需要：

### FastAPI / runtime 最小集合

```text
NPC_BRAIN_DEPLOY_API_KEY=<shared-key>
```

### 若要測 Gemini

```text
GOOGLE_API_KEY=<key>
NPC_LLM_PROVIDER_ORDER=gemini,gemini_flash,gemini_flash_lite,local_llama,history_cache,deterministic
```

### 若要測 Pinecone

```text
PINECONE_API_KEY=<key>
NPC_PINECONE_INDEX=sanguo-npc-brain-dev
```

## 啟動 FastAPI dev server

```bash
cd server/npc-brain
"$PYTHON_BIN" -m uvicorn app.main:app --host 127.0.0.1 --port 8765 --reload
```

## 啟動 LangGraph dev server

```bash
cd server/npc-brain
"$PYTHON_BIN" -m langgraph dev --no-browser
```

## 先看健康檢查

```bash
curl http://127.0.0.1:8765/healthz
```

如果有 vector 設定，也會在 `vector` 區塊看到摘要。

## 常見狀況

### `pinecone.configured = false`

通常是 `.env` 沒有正確載入 `PINECONE_API_KEY`。

### `qdrantPing.reachable = false`

通常是 Docker daemon / Qdrant container 沒起來。

### localtunnel 或 deploy 能進，但 dialogue 失敗

通常是：

- provider 設定缺失
- readiness 資產不完整
- `generalId` / `contextKey` / `selectedKeywordKeys` 不匹配
