# 最短啟動路徑

> 文件使用原則：
> - 本文假設你正在使用 standalone `3klife-npc-brain` repo。
> - `<repo-root>` 代表你自己 clone 下來的專案根目錄。
> - 正式開發環境以 Docker 為主；本機 Python / venv 只在你需要 IDE debug 或 LangGraph dev 時才使用。

這份 runbook 給第一次接手的人用：目標不是理解全部細節，而是先把 NPC brain 服務在本機穩定跑起來。

## 你只要做這 5 步

### 1. 進 repo

```powershell
cd <repo-root>
```

### 2. 準備 `.env`

```powershell
Copy-Item .env.example .env
```

如果你暫時只想把服務跑起來，至少先補：

```text
NPC_BRAIN_DEPLOY_API_KEY=<shared-key>
GOOGLE_API_KEY=<key>
NPC_LLM_PROVIDER_ORDER=deterministic
```

## 3. 啟動 Docker 開發環境

```powershell
docker compose -f docker-compose.dev.yml up -d --build
```

## 4. 確認服務活著

```powershell
curl http://127.0.0.1:8765/healthz
docker compose -f docker-compose.dev.yml ps
```

你想看到的狀態是：
- `/healthz` 有回應
- `3klife-npc-brain-dev` 是 `healthy`

## 5. 跑最小 smoke test

```powershell
docker exec 3klife-npc-brain-dev python -B -m app.http_smoke_test
docker exec 3klife-npc-brain-dev python -B pipelines/sanguo-rag/validate_sanguo_governance.py --dry-run-report
```

這兩個都過，代表：
- FastAPI 入口正常
- Sanguo governance 資料沒有明顯壞掉

## 接下來怎麼用

- 3KLife / Cocos 端預設打 `http://127.0.0.1:8765`
- 主要 API：
  - `GET /healthz`
  - `GET /v1/npc/context-options`
  - `GET /v1/npc/keyword-options`
  - `POST /v1/npc/dialogue`

## 要停掉服務

```powershell
docker compose -f docker-compose.dev.yml down
```

## 什麼時候才需要本機 Python / venv

只有這幾種情況才需要：
- 你要用 IDE 逐步 debug Python
- 你要跑 `langgraph dev --no-browser`
- 你要臨時執行某支不方便放進 container 的工具

其餘情況，優先用 Docker，就最穩。
