## Project Architecture

### High-Level Structure

**3klife-npc-brain** is a modular NPC dialogue service split into three layers:

1. **FastAPI Service Layer** (`app/`)
   - HTTP endpoints: `/healthz`, `/v1/npc/dialogue`, `/v1/npc/context-options`, `/v1/npc/keyword-options`, `/v1/npc/narrative-profile`, `/v1/npc/memory/*`
   - `NpcDialogueService`: Core orchestrator that ties LangGraph graphs, vector search, and LLM rendering
   - `LLMDialogueRenderer`: Handles multi-provider LLM generation (Gemini, Qwen, DeepSeek, local Llama) with fallback chains
   - `InteractionMemory`: Stores and compresses game event history
   - `VectorStore` + `VectorEmbedding`: Pinecone/Qdrant integration for semantic retrieval

2. **LangGraph Workflow Layer** (`langgraph_app/`)
   - `graph.py`: NPC Studio graph for interactive dialogue generation (used in LangGraph Studio)
   - `etl_graph.py`: Data pipeline orchestration for persona, keyword, relationship generation
   - `etl_repair_graph.py`: Data validation and repair workflows
   - `progress_advancement_graph.py`: Pipeline state machine for multi-round knowledge growth

3. **Sanguo-RAG Pipeline** (`pipelines/sanguo-rag/`)
   - Generates persona cards, keyword options, event candidates, and relationship graphs
   - Multi-stage: evidence collection ??cleaning ??entity extraction ??LLM reasoning ??vector indexing ??governance validation
   - Key artifacts output to `artifacts/data-pipeline/sanguo-rag/`
   - Heavy use of external sources (3kweb, Novel data, relationship databases)
   - Includes governance validators to catch schema violations, missing fields, semantic conflicts

### Config & State Management

- **Environment**: `.env` (copy from `.env.example`); key vars: `NPC_BRAIN_DEPLOY_API_KEY`, `GOOGLE_API_KEY`, `NPC_LLM_PROVIDER_ORDER`
- **Vector Config**: `app/vector_config.py` (Pinecone/Qdrant endpoint, index names)
- **Runtime Profiles**: Generated by Sanguo pipeline, stored in `artifacts/data-pipeline/sanguo-rag/runtime-profiles/`
- **Postgres DSN**: `SANGUO_RAG_PG_DSN` points to state store (migrations managed separately)
- **ATM Governance**: `.atm/runtime/` holds pinned configs, validated policy, and project probe

## API Contract

### Core Endpoints (Cocos Integration)

| Method | Path | Purpose | Example |
|--------|------|---------|---------|
| GET | `/healthz` | Service liveness | Returns `{"status":"ok"}` |
| GET | `/v1/npc/context-options?generalId=zhang-fei` | Retrieves contextual memory snippets for an NPC | Pagination + filtering by generalId |
| GET | `/v1/npc/keyword-options?generalId=zhang-fei&categories=person,item` | Retrieves keyword options (persona, relationships, items) | Supports category filtering + per-category limits |
| POST | `/v1/npc/dialogue` | Generate NPC dialogue | Input: `DialogueRequest` (generalId, keywords, locale, model), Output: `DialogueResponse` (text, provider, model) |
| GET | `/v1/npc/narrative-profile?generalId=zhang-fei` | Full persona card and metadata | Rich persona snapshot for UI rendering |

### Memory Endpoints

| Method | Path | Purpose |
|--------|------|---------|
| POST | `/v1/npc/interaction-events` | Log game events (dialogue, combat, discoveries) |
| GET | `/v1/npc/general-memory` | Retrieve compressed memory state |
| POST | `/v1/npc/general-memory` | Save memory checkpoint |
| POST | `/v1/npc/memory/compress` | Compress long memory chains |

## Integration Points

### Vector Search
- **Pinecone/Qdrant**: Stores embeddings of persona cards, keywords, events, relationships
- **Flow**: Pipeline generates artifacts ??Embeddings computed ??Vectors upserted ??Dialogue service queries

### LLM Providers (Configured Fallback Chain)
- **Primary**: Gemini Flash (Google)
- **Secondary**: Qwen 2.5 (Alibaba)
- **Tertiary**: DeepSeek R1
- **Fallback**: Local Llama (if configured)
- **Error Handling**: Provider unavailable ??fallback chain auto-triggers; full chain exhausted ??503

### Cocos Integration
- Cocos client calls `http://<npc-brain-host>:8765/v1/npc/*` endpoints
- No direct Python imports; communication is REST only
- Game saves dialogue history + events via `/v1/npc/interaction-events`

## Common Workflows

### Debug Dialogue Generation
1. Check LLM provider health: Smoke tests in `app/llm_provider_smoke_test.py`
2. Verify vector search: `app/vector_second_smoke_test.py` (queries Pinecone/Qdrant)
3. Inspect dialogue state: Add logging to `app/llm_dialogue_renderer.py`, re-run `app.http_smoke_test`

### Add New NPC or Update Persona
1. Update Sanguo pipeline inputs (novel data, relationships) in `pipelines/sanguo-rag/`
2. Run full ETL: `docker exec ... python -B langgraph_app/etl_graph.py`
3. Validate output: `validate_sanguo_governance.py --dry-run-report`
4. Restart FastAPI container to reload artifacts

### Regression Testing
- Full pipeline validation: `run_sanguo_governance_regression_harness.py --run-profile strict-local --no-write`
- Dialogue smoke test: `app.http_smoke_test` (tests top ~15 popular generals)

## Troubleshooting

| Symptom | Root Cause | Solution |
|---------|-----------|----------|
| `/healthz` fails | Service not running or unhealthy | Check Docker container logs: `docker logs 3klife-npc-brain-dev` |
| Dialogue returns 503 | All LLM providers unavailable | Check `.env` LLM keys (GOOGLE_API_KEY, QWEN_API_KEY, etc.) |
| Vector search empty results | Pinecone/Qdrant not indexed | Rebuild index: run Sanguo pipeline ETL stage |
| Missing persona data | Artifacts not generated or outdated | Re-run `pipelines/sanguo-rag/run_sanguo_governance_regression_harness.py` |
| LangGraph Studio hangs | Graph definition or state type error | Check `langgraph_app/*.py` for TypedDict/node definition issues |
